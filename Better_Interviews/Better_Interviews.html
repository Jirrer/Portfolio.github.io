Description

Better Interview is a web-based application designed to help job seekers track and prepare for upcoming interviews. The app provides a clear visual overview of your interview progress using three circle graphs that display the ratio of accepted, pending, and rejected interviews.

Beyond tracking, Better Interview also helps with preparation. When a new interview is created, the backend sends the interview details to a Gemini AI prompt, which generates a personalized study plan tailored to the position, company, and relevant skills.

How-To

To use Better Interview:

  1. Log in to your account or sign up if you are a new user.
  2. Click “Add New Interview” and enter the required details, including position, company, and interview date.
  3. Once submitted, the app will display your interview in the dashboard and automatically generate a study plan using Gemini AI.
  4. Use the circle graphs to quickly see your interview status breakdown (Accepted, Pending, Rejected).

Documentation

Prerequisites

Better Interview requires the following before it can run:

  • Node.js 16 or higher - Required for backend operations and Gemini API calls.
  • A modern web browser - For the best experience, Chrome or Edge is recommended.
  • Gemini API key - Used to generate personalized interview study plans.
  • Database connection - The app uses a database to store interviews and their statuses.

Configuring Environment Variables

All required keys and configuration values must be stored in the .env file located in the project root. Example: 

    GEMINI_API_KEY=your_gemini_key_here
    DATABASE_URL=your_database_connection_string

Keep this file private to ensure sensitive credentials are not exposed.

Frontend

The frontend is built with HTML, CSS, and JavaScript, providing a responsive layout for both desktop and mobile use. The dashboard prominently features three animated circle graphs showing the distribution of interview statuses.

Backend

Interview Tracking

The backend stores all interview data, including the position, company name, date, and status. This data is used to calculate the ratios displayed in the frontend circle graphs.

Gemini Integration

When a new interview is added, the backend sends the interview details to a Gemini AI prompt. Gemini processes the information and returns a tailored study plan, which may include company research topics, common role-specific interview questions, and suggested practice exercises.

Database

The database stores all interviews with their associated metadata. Status updates trigger the frontend graphs to refresh, ensuring real-time accuracy.
